# 딥 리서치 Skill

어떤 주제를 조사하라는 요청을 받으면 다음 방법론을 따르세요:

## 조사 도구 (우선순위)

1. **브라우저 도구** (우선 —— 웹 조사에서 가장 신뢰할 수 있음):
   - `browser_navigate(url:)` → 앱 내 브라우저에서 URL 열기
   - `browser_get_page_info(include_content: true)` → 페이지 텍스트, 링크, 상호작용 요소 읽기
   - `browser_extract(selector:)` → 특정 요소(제목, 본문 등) 추출
   - **검색 전략**: `https://www.google.com/search?q=YOUR+QUERY` 로 이동해 출처를 찾고, 유망한 결과를 방문
2. **`fetch_and_extract`** (경량 평문 가져오기 —— 샌드박스 네트워크 제한으로 실패할 수 있음. HTTP 0 또는 네트워크 오류 시 즉시 브라우저 도구로 전환)
3. **후처리 스크립트** (`run_snippet` 으로 실행):
   - `extract_links` —— HTML 에서 후속 URL 을 추출하여 심화 조사
   - `summarize_text` —— 긴 텍스트를 중요도 순으로 요점화

## 프로세스
1. **분해** —— 질문을 2~5 개의 집중 하위 질문으로 나눔
2. **검색** —— 브라우저의 검색 엔진에서 시작해 가장 유망한 결과 방문
3. **수집** —— 각 출처를 `browser_get_page_info` 로 읽고, 특정 내용은 `browser_extract`, 링크를 따라 심화
4. **평가** —— 출처 신뢰도(공식 문서 > 논문 > 권위 있는 언론 > 블로그), 최신성, 편향 가능성 평가
5. **종합** —— 여러 출처를 상호 참조해 일관된 분석으로 정리
6. **인용** —— 주요 주장마다 출처 URL 과 신뢰도(높음 / 중간 / 낮음) 명시

## 반복적 심화
- 첫 라운드 후 누락 또는 상충되는 정보 식별
- 추가 출처를 검색해 누락을 채우고 상충을 해결
- 주요 주장에 2~3 개의 상호 근거가 확보되거나, 추가 출처가 더 이상 새로운 정보를 주지 않을 때 종료

## 출력 형식
- **요약** —— 2~3 문장으로 된 주요 발견 개요
- **발견** —— 하위 주제별 구성, 출처 인용을 인라인으로 포함
- **출처 표** —— 사용한 출처와 신뢰도(높음 / 중간 / 낮음) 목록
- **불확실성** —— 상충 정보와 지식 공백 표시
- **결론** —— 실행 가능한 핵심 내용

## 가이드라인
- 1 차 출처(공식 문서, 논문, 원자료)를 2 차(뉴스, 블로그)보다 우선
- 정보가 오래되었을 수 있음에 유의 —— 게시 날짜 확인
- 확정된 사실, 전문가 의견, 추측을 명확히 구분
- 도구가 실패하면 즉시 대체로 전환 —— 동일한 실패 도구를 재시도하지 말 것
- 주요 발견은 MEMORY.md 에 저장해 이후 참조
