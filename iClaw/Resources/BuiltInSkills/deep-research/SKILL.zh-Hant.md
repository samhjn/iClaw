# 深度研究 Skill

當被要求研究某個主題時，依照以下方法論執行：

## 研究工具（依優先順序）

1. **瀏覽器工具**（首選 —— 網頁研究最可靠）：
   - `browser_navigate(url:)` → 在內建瀏覽器中開啟 URL
   - `browser_get_page_info(include_content: true)` → 讀取頁面文字、連結與互動元素
   - `browser_extract(selector:)` → 擷取特定元素（標題、正文等）
   - **搜尋策略**：前往 `https://www.google.com/search?q=YOUR+QUERY` 找來源，再訪問有潛力的結果
2. **`fetch_and_extract`**（輕量純文字抓取 —— 可能因沙盒網路限制失敗；若回傳 HTTP 0 或網路錯誤，立刻切換至瀏覽器工具）
3. **後處理指令稿**（以 `run_snippet` 執行）：
   - `extract_links` —— 從 HTML 擷取後續 URL 以深入調查
   - `summarize_text` —— 將長文字濃縮為依重要性排序的要點

## 流程
1. **拆解** —— 將問題拆為 2–5 個聚焦的子問題
2. **搜尋** —— 從瀏覽器中的搜尋引擎開始，訪問最有潛力的結果
3. **蒐集** —— 對每個來源使用 `browser_get_page_info`；以 `browser_extract` 擷取特定內容；沿著連結深入
4. **評估** —— 評估來源可信度（官方文件 > 論文 > 知名媒體 > 部落格）、時效與潛在偏誤
5. **綜合** —— 將發現整理為連貫分析，跨來源交叉佐證
6. **引用** —— 每個關鍵主張皆註明其來源 URL 與信心度（高 / 中 / 低）

## 反覆深化
- 第一輪後，辨識缺口或衝突資訊
- 搜尋額外來源填補缺口或解決衝突
- 當關鍵主張有 2–3 個來源相互佐證，或新增來源不再帶來新資訊時停止

## 輸出格式
- **執行摘要** —— 2–3 句關鍵發現概述
- **發現** —— 依子題組織，附內嵌來源引用
- **來源表** —— 列出使用的來源與可信度（高 / 中 / 低）
- **不確定性** —— 標註衝突資訊與知識缺口
- **結論** —— 可操作的要點

## 準則
- 偏好一手來源（官方文件、論文、原始資料）而非二手（新聞、部落格）
- 注意資訊可能過時 —— 檢查發佈日期
- 清楚區分既定事實、專家觀點與推測
- 工具失敗時立即切換替代方案 —— 不要重試同一個失敗的工具
- 將關鍵發現保存至 MEMORY.md 以供日後參考
