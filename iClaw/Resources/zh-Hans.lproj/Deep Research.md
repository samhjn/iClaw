# 深度研究 Skill

当被要求研究某个话题时，按以下方法论执行：

## 研究工具（按优先级）

1. **浏览器工具**（首选 —— 网页研究最可靠）：
   - `browser_navigate(url:)` → 在内置浏览器中打开 URL
   - `browser_get_page_info(include_content: true)` → 读取页面文本、链接与可交互元素
   - `browser_extract(selector:)` → 提取特定元素（标题、正文等）
   - **搜索策略**：访问 `https://www.google.com/search?q=YOUR+QUERY` 找到来源，然后访问可能的结果页
2. **`fetch_and_extract`**（轻量纯文本抓取 —— 可能因沙盒网络限制失败；如果返回 HTTP 0 或网络错误，立刻切换到浏览器工具）
3. **后处理脚本**（用 `run_snippet` 执行）：
   - `extract_links` —— 从 HTML 中提取后续 URL 以深入调查
   - `summarize_text` —— 将长文本浓缩为按重要性排序的要点

## 流程
1. **拆解** —— 将问题拆成 2–5 个聚焦的子问题
2. **搜索** —— 从浏览器里的搜索引擎开始，访问最有潜力的结果
3. **收集** —— 对每个来源用 `browser_get_page_info`；用 `browser_extract` 抓特定内容；顺着链接深入
4. **评估** —— 评估来源可信度（官方文档 > 论文 > 知名媒体 > 博客）、时效性与潜在偏差
5. **综合** —— 把发现整理成连贯分析，跨来源交叉印证
6. **引用** —— 每个关键结论都标注其来源 URL 及置信度（高 / 中 / 低）

## 迭代深化
- 第一轮后，识别空白或冲突信息
- 搜索额外来源填补空白或解决冲突
- 当关键结论有 2–3 个来源相互印证，或增加来源已无新信息时停止

## 输出格式
- **执行摘要** —— 2–3 句关键发现概述
- **发现** —— 按子主题组织，附内嵌来源引用
- **来源表** —— 列出使用的来源及可信度（高 / 中 / 低）
- **不确定性** —— 标注冲突信息与知识空白
- **结论** —— 可操作的要点

## 指南
- 优先一手来源（官方文档、论文、原始数据）而非二手（新闻、博客）
- 注意信息可能过时 —— 检查发布日期
- 清晰区分既定事实、专家观点与推测
- 工具失败时立即切换替代方案 —— 不要重试同一个失败的工具
- 将关键发现保存到 MEMORY.md 以便后续参考
